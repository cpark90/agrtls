# CODESTYLE.md
이 코드 스타일은 상세설계 및 구현 시, 매 세션 시작 시 읽는다.

## 1. 공통 철학 (언어 무관)

Google 가이드의 근본 목표는 **일관성과 가독성**이다. "다른 사람의 코드를 보고 빠르게
이해할 수 있어야 한다"가 모든 규칙의 이유다.

- **[지킴]** 일관성이 최우선. 한 파일·한 모듈 안에서 스타일을 섞지 않는다. 기존 코드의
  지역 컨벤션이 있으면 그것을 따른다.
- **[지킴]** 코드는 자기설명적으로. 좋은 이름이 주석보다 낫다. 주석은 "무엇"이 아니라
  "왜"를 적는다.
- **[지킴]** 함수는 작고 한 가지에 집중. 하드 리밋은 없지만 **약 40줄을 넘으면** 쪼갤지
  검토한다(Google C++·Python 공통 권고).
- **[지킴]** 중첩 대신 가드절. 일찍 `return`/`continue` 하고 깊은 중첩을 피한다.
- **[권장]** 복잡한 식은 이름 있는 중간 변수로 쪼갠다(`result = (a+b)*(c-d)/(e+f)` 대신
  단계로 나눈다).
- **[권장]** 영리하거나 특이한 구성을 경계한다. "금지되지 않았다"가 "써도 된다"는 아니다.
- **[권장]** 매직넘버 대신 이름 있는 상수를 쓴다.
- **[지킴]** 들여쓰기는 **스페이스만**. 탭을 쓰지 않는다(C++ 2칸 / Python 4칸 — 아래 참조).
- **[지킴]** 한 줄 길이는 **80자**가 기준. 넘으면 끊어 쓴다(팀 합의 시 Python은 99자까지 허용).

## 2. C++ (Google C++ Style Guide)

### 포맷
- **[지킴]** 들여쓰기 **스페이스 2칸**. 탭 금지.
- **[지킴]** 한 줄 **80자**.
- **[지킴]** 여는 중괄호 `{`는 같은 줄에. 함수명과 `(` 사이 공백 없음(`Func(arg)`).
- **[권장]** 함수 인자가 한 줄에 안 들어가면 첫 인자에 정렬하거나 4칸 들여 줄바꿈.
- **[권장]** 연산자 양옆·콤마 뒤에 공백. `.`/`->`/`++`/`--` 주위에는 공백 없음.
- **[권장]** 함수/클래스 사이는 빈 줄 1개(또는 몇 개)로 단락처럼 구분.

### 네이밍 (Google C++의 핵심 — 42 Norm과 다름)
- **[지킴]** 함수: `PascalCase` — `void SendReading();`, `float GetRange();`
- **[지킴]** 변수(지역·인자): `snake_case` — `int sample_count;`
- **[지킴]** 클래스 멤버 변수: `snake_case_` 끝에 언더스코어 — `float last_range_;`
- **[지킴]** 타입(클래스/구조체/enum/typedef): `PascalCase` — `class AnchorTrack;`
- **[지킴]** 상수·`constexpr`: `kCamelCase` — `constexpr int kMaxAnchors = 8;`
- **[지킴]** 매크로: `ALL_CAPS` — `#define ROLE_TAG`
- **[지킴]** 네임스페이스·파일명: `snake_case`. 파일은 소문자, `.h`/`.cc`(임베디드는 `.cpp` 허용).

### 구조·관용구
- **[지킴]** 헤더 가드 `#ifndef PROJECT_PATH_FILE_H_` 또는 `#pragma once`.
- **[지킴]** include 순서: 관련 헤더 → C 시스템 → C++ 표준 → 기타 라이브러리 → 프로젝트.
- **[지킴]** 인자는 선언·정의에서 동일한 이름. 안 쓰는 인자는 이름을 주석 처리(`/*radians*/`).
- **[권장]** 변수는 가장 좁은 스코프에 선언하고 **선언과 동시에 초기화**.
- **[권장]** `++i`를 `i++`보다 선호(루프 카운터).
- **[권장]** 크기 보장이 필요한 정수는 `<cstdint>`의 `int16_t`/`uint32_t` 등을 쓴다.
- **[권장]** 멤버 데이터는 기본 `private`, 접근은 getter/setter. `public`은 필요할 때만.
- **[지킴]** 전역 네임스페이스 오염 금지. `using namespace ...` 금지.
- **[권장]** RAII로 자원 관리. raw `new`/`delete` 대신 스택 객체·`unique_ptr`.
- **[권장]** 다형성이 실제로 필요할 때까지 `virtual`·상속·템플릿을 만들지 않는다.

## 셀프체크

작업 완료 전 해당 언어 도구로 점검한다. 규칙을 어길 땐 **[지킴]/[권장]** 항목에 한해
그 줄에 사유를 한 문장 남긴다 — 말없이 머지하지 않는다.
